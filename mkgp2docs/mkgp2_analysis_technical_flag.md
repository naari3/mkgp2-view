# 解析報告: g_technicalTitleGrade (称号テクニカルフラグ) の調査結果

このドキュメントは、MKGP2 における称号判定フラグ `0x806cf1fc` (`g_technicalTitleGrade`) の役割と変遷条件についての詳細な解析結果をまとめたものです。

## 1. 概要
`g_technicalTitleGrade` は、レース中のコイン/エフェクトターゲットの回収状況をリアルタイムで評価し、最終的な称号（TitleID 136-138）を決定するための「階層的ステータスフラグ」です。

## 2. 各値の条件と意味

| 値 | 称号 ID (raw) | 最終 TitleID (+45) | 称号名 (実機確認済) | 発生条件概要 |
|:---:|:---:|:---:|:---|:---|
| **2** | 93 (0x5d) | **138** | **噂の (Rumored)** | コイン出現後、残存数が一度も変化しなかった（1枚も回収/消滅なし）+ 無敗 |
| **1** | 92 (0x5c) | **137** | **パーフェクト (Perfect)** | 出現総数 == 最終残存数（結果的に回収ゼロ）。通常走行のフォールバック |
| **0** | 91 (0x5b) | **136** | **超一流の (Top)** | 全エフェクトイベントを漏れなく処理（全回収） |
| **-1** | なし | - | なし | 初期状態、またはコイン残存数変化によるリセット |

---

## 3. kartData + 0x2cc の正体

### 旧解釈（誤り）
以前の解析では `kartData + 0x2cc` を「driveState（走行ステート: 直進、ドリフト、ジャンプ等）」と解釈していたが、**これは誤りである**。

### 新解釈（確定）
`kartData + 0x2cc` は**残存エフェクトターゲット数**（コイン等の残存カウンタ）。

#### 根拠
1. **FUN_80045b50** (KartData初期化): `param_1[0xb3] = 0` (0xb3 * 4 = 0x2cc) → 0 で初期化
2. **FUN_8013a5bc** (エフェクト更新): 生存エントリを数えて `*(param_1 + 0x2cc) = survivingCount` に格納
3. **FUN_8004a238 / FUN_8004a918** (ヒットハンドラ): `+0x2cc` を -1 デクリメント（clamp to 0）
4. **FUN_800a2b90** (メインループ): 2Pの `+0x2cc` をインクリメント。表示関数 FUN_80253e18 にも渡す

---

## 4. 詳細な変遷ロジック

### A. 初期化と失効 (Reset/Abort)
- **初期化**: `RankLog_InitStartRace` (0x800a67a8) にて、レース開始時に `-1` (0xffffffff) がセットされる。
- **リセット条件**: `RankLog_UpdatePerFrame` (0x800a59f0) にて毎フレーム監視。
  - **ロジック**: 前フレームの残存数 (`DAT_806d13f0`) と現在の残存数 (`kartData + 0x2cc`) が異なり、かつ現在のグレードが `2` (「噂の」) である場合、強制的に `-1` へ戻される。
  - **意味**: 「噂の」状態を維持するには、コイン残存数が変化してはならない。コイン回収やアイテムヒットによる消滅が1回でもあると即失効。

### B. 「2: 噂の」への昇格 (最高グレード)
- **判定関数**: `RankLog_UpdateTechnicalFlag` (0x800a5e60)
- **呼び出し元**: `FUN_8013a778` (エフェクト管理関数)
- **トリガー**: コイン (`s_item_coin_dat`)、マリオコイン、キラキラエフェクト (`s_k_kira13_dat`) 等が出現した際。
- **条件**: 出現した有効エフェクト数 (`DAT_806d1538`) が 1 以上であること。
- **処理**: グレードを `2` にセットし、エフェクト数を `DAT_806d13ec` に保存。また `RsInfomation` の総イベント数 (`+0x1c`) を更新。

### C. 「1: パーフェクト」への昇格
- **判定関数**: `RankLog_SetRaceResult` (0x800a5cec)
- **タイミング**: レース終了時（`FUN_800a5da0` より先に呼ばれる）
- **条件**: `DAT_806d13ec == DAT_806d13f0`
  - `DAT_806d13ec`: エフェクト出現総数（`RankLog_UpdateTechnicalFlag` で記録）。
  - `DAT_806d13f0`: 最終的な残存エフェクトターゲット数（`RankLog_UpdatePerFrame` で毎フレーム更新）。
- **意味**: 出現総数と最終残存数が一致 → 結果的にコインを1枚も回収しなかった状態。
- **注意**: この後に `FUN_800a5da0` が呼ばれるため、全回収条件も満たしていればグレード0で上書きされる。

### D. 「0: 超一流」への昇格
- **判定関数**: `FUN_800a5da0` (0x800a5e38付近)
- **タイミング**: レース終了時（`RankLog_SetRaceResult` の後に呼ばれる）
- **条件**: `RsInfomation` (`FUN_8024f588`) のメンバ比較:
  - `総イベント数 (+0x1c)` == `処理済みイベント数 (+0x18)`
  - かつ `phase == 1` (FUN_800a2b90 から `DAT_806d1344 - 1` が渡される)
- **前提**: `DAT_806d13ec > 0` (何らかのエフェクトが発生していること)。
- **意味**: 出現した全てのエフェクトイベントを取りこぼしなく回収/処理した状態。グレード1を上書きする。

---

## 5. 称号判定への統合
レース終了時の `determine_title` (0x800a70fc) から、各チェック関数がこのフラグを評価する。

1. **FUN_800a7e38 (Race[11])**: グレード `0` → **136 (超一流の)**
2. **FUN_800a7dd4 (Race[12])**: グレード `2` かつ無敗 → **138 (噂の)**
3. **FUN_800a7e0c (Race[21])**: グレード `1` → **137 (パーフェクト)**

※ グレード1はテーブル上の優先度が最も低い (index 21)。グレード0や2が先にマッチするため、「通常走行」（コインに触れなかった場合）のフォールバック的な称号。

## 6. グレード1とグレード2の違い

一見どちらも「コイン未回収」だが、判定タイミングが異なる:

- **グレード2**: レース中リアルタイム監視。`kartData + 0x2cc` が1回でも変化した瞬間に失効。コイン回収だけでなく、アイテムヒットによるコイン消滅でも失効する。
- **グレード1**: レース終了時の最終判定のみ。途中で増減があっても、最終的に出現総数 == 残存数なら成立する可能性がある。

## 7. 結論
`g_technicalTitleGrade` は、コイン/エフェクトターゲットの回収状況を評価する変数である。**「全回収」** (グレード0 = 超一流) と **「未回収」** (グレード1 = パーフェクト / グレード2 = 噂の) という対照的な条件で称号が決まる。特に「噂の」称号は、コイン出現後に残存数が一切変化しないという条件に加え、無敗であることも要求される。
