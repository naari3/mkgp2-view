# MKGP2 カートチューニングテーブル完全ダンプ

## テーブル構造

- **ベースアドレス**: `0x8049D030` (RACE mode)
- **インデックス式**: `(courseId-1) * 0xC0 + ccClass * 0x40 + subMode * 8 + kartSlot`
- **エントリサイズ**: 0x17 words = 0x5C bytes (92 bytes)
- **パラメータ**: courseId=1-8, ccClass=0-2, subMode=0 (メインモード), kartSlot=0-7
- **kartSlot**: カートバリアント (NORMAL/ORIGINAL/ENHANCED) ではなく、レース中のスロット番号。全スロットAI用 (プレイヤーの速度はこのテーブルに影響されない)。Slot 0=最速AI、Slot 4=最遅AI
- **テーブル終端**: `0x804BE410` (8*3*8 entries * 0x5C = 0x21DE0 bytes)

## BaseSpeed テーブルとの関係

チューニングテーブルの `speedFactor` は、BaseSpeed テーブル (0x803A01E8) から取得したベース速度に対する係数として使われる。

- **BaseSpeed テーブル**: `GetBaseSpeedMax/Min(courseId, subMode)` で AI のベース速度 (speedMin, speedMax) を取得
  - インデックス: `(courseId - 1) * 0x18 + ccClass * 8 + subMode`
  - subMode 0-3 = 表コース、subMode 4-7 = 裏コース
  - 裏コースは表コースよりベース速度が高いため、同じ speedFactor でも AI の実効速度が上がる
- **チューニングテーブル**: speedFactor を乗じた結果が AI の目標速度になる

つまり、裏コースで AI が速い原因はチューニングテーブル側ではなく、BaseSpeed テーブルの subMode 別エントリにある。チューニングテーブル自体の subMode 間差異は主に `aiPattern` (走行パターン) のみ。

## 周回ボーナス (AI_CalcLapSpeedBonus)

チューニングテーブルとは別に、**残りラップ数に応じた速度ボーナス**が BaseSpeed に加算される。
AI_CalcLapSpeedBonus (0x801dd480) がコース別のルールテーブルを検索して決定する。

**注: 以前「順位ボーナス」と誤認していたが、正しくは周回ボーナス。** 引数は `totalLaps - currentLap`（残りラップ数）であり、レース順位ではない。レース順位は一部ルールの除外条件としてのみ使用される。

### マリオカップ 150cc (9ラップ) の例

| 残りラップ | ボーナス | 効果 |
|---|---|---|
| 0 (最終ラップ) | **+60.0** | BaseSpeed≈270 に対して +22% |
| 1 | +20.0 | +7.4% |
| 2-3 | +10.0 | +3.7% |
| 4-7 | +0.0 (ルールなし) | ボーナスなし |
| 8 (序盤, lapDiff 50+) | -15.0 | ペナルティ |

- **レース終盤ほどAIが速くなるラバーバンド機構**
- 150cc全コース共通: 最終ラップで +60.0 (lapDiff 50以内)
- 最終ラップにはさらにスロット別・順位除外付きの追加ボーナスあり
- subMode (0-7) で難易度が異なり、高subModeほど攻撃的なボーナス設定

詳細: `mkgp2docs/mkgp2_lap_speed_bonus.md`

### S0-S3 が距離を取りすぎた場合

明示的な減速ペナルティはない。起きること:
1. チューニングテーブルの aheadSpd (+10〜20) が距離閾値超過で消える → **軽微な減速**
2. 周回ボーナスは残りラップ数に依存するため、距離では変化しない → **減速しない**
3. AI_CalcDistanceSpeedBonus (0x801dcdcc) にコース別ルールがあるが、通常条件で負値は確認されない

結論: S0-S3 は先行しすぎても大幅には遅くならない。最後尾AIの周回遅れは speedFactor/accelRate/steeringDamping の差が原因。

## フィールド定義 (23 words)

### テーブル列の略称

データテーブル内の列ヘッダーは以下の略称を使用:

| 略称 | フィールド名 | 説明 |
|------|-------------|------|
| **speed** | speedFactor | AI目標速度の係数 |
| **accel** | accelRate | 入力スムージングの最大変化率 |
| **f02** | (未参照) | コード上参照なし |
| **f03** | (未参照) | コード上参照なし |
| **aD** | aheadDistThreshold | GP AI先行判定の距離閾値 |
| **aSpd** | aheadSpeedBonus | GP AI先行時の速度ボーナス |
| **bD** | behindDistThreshold | GP AI後方判定の距離閾値 |
| **bSpd** | behindSpeedBonus | GP AI後方時の速度調整 (負=減速) |
| **timer** | speedAdjustTimer | 速度調整タイマーの基準値 |
| **aiV** | aiVtableSelector | AI行動パターンのvtable選択 |
| **steer** | steeringDamping | ステアリング力の減衰係数 |
| **i13-i19** | aiPattern[0-6] | AI行動パターン配列 |

| Word | Offset | 名前 | 型 | 値域 | 参照元コード | 詳細説明 |
|------|--------|------|-----|------|-------------|---------|
| 0 | +0x00 | speedFactor | float | 0.0~0.6 | `GetKartTuningSpeedFactor` (0x801de4d4) | AI目標速度の係数。BaseSpeed テーブル (0x803A01E8) の `GetBaseSpeedMax/Min(courseId, subMode)` から取得したベース速度にこの値を乗じる。プレイヤー (Slot 0) が最高値 0.6、AIスロットが大きいほど低下 |

| 9 | +0x24 | behindSpeedBonusExtra | float | 常に0 | `AI_CalcGPTargetSpeed` | 後方時追加調整。`[8]` に加算。全データで 0.0 |

| 20-22 | +0x50~+0x58 | (unused) | int | 常に0 | — | 未使用パディング |

## スロット分類

> **注意**: 旧ドキュメントではこのインデックスを「カートバリアント (NORMAL/GOLD/ORIGINAL/ENHANCED)」と
> 誤記していたが、実際は**カートスロット番号**である。`FUN_801ebc8c` でループインデックス (0-7) が
> `PhysicsState+0x10` に格納され、以降 `GetKartTuningEntry` の引数として使われる。

| Slot | 旧名称 (誤) | 正しい意味 | 特徴 |
|------|------------|-----------|------|
| 0 | NORMAL | **プレイヤー** | speedFactor最高、accelRate高、timer=180 |
| 1 | GOLD | **AI 1番** | aiVtbl=1。behindDist=32固定 |
| 2 | ORIGINAL | **AI 2番** | timer短め (30~50)。behindDist=24固定 |
| 3 | ENHANCED | **AI 3番** | aiVtbl=1。accelRate低、timer短め |
| 4 | S4(AI) | **AI 4番** | 最弱。accel=0.5、steerDamp=5 |
| 5-7 | V5-V7 | **未使用** | ゼロ埋め、i13-i15=-1 sentinel |

## AI行動パターン配列 (i13-i19) の分析

i13-i19 は値範囲 0-9 の整数7個の配列。コースごとに固定パターンで、CCクラスでは変化しない。
スロット間では異なるパターンが割り当てられている。おそらくAIのアイテム使用優先度や走行ルート選択を制御。

### 特殊ケース
- **Mario 50cc / Yoshi 50cc**: i13-i19 = 全て0 (初期状態のまま?)
- **V5-V7 (全コース)**: i13-i15 = -1 (0xFFFFFFFF) -- 未使用sentinel

### i16-i18 の固定パターン (スロット依存)

スロット間で i16-i18 はほぼ固定:
- S0 (Player): i16=3, i17=6, i18=1
- S1 (AI): i16=4, i17=7, i18=2
- S2 (AI): i16=5, i17=8, i18=3
- S3 (AI): i16=3, i17=9, i18=1
- S4 (AI): i16=4, i17=8, i18=2

(Mario Cup のみ若干のCC依存バリエーションあり)

### i19 の値分布

- **DK/Wario/Pac-Man/Bowser**: i19 = 9 (全スロット)
- **Rainbow/Yoshi(100+)/Waluigi**: i19 = 5 (S0/S2/S4), 6 (S1/S3)
- **Mario**: i19 = 3 (S0/S2/S4), 8 (S1/S3)
- **Mario/Yoshi 50cc**: i19 = 0 (全スロット = 未設定)

## 分析まとめ

### 独自チューニングを持つコース

1. **Mario Cup (c=1)**: 全CC で speedFactor 非ゼロ (0.2~0.6)。50cc で behindSpd=-5、100cc で behindSpd=+5。f02 は CC に応じて 20→20→100 と増加。AI パターン (i13-i19) は CC ごとに変動。50cc のみ i13-i19=全0 (未設定)。
2. **Yoshi Cup (c=7)**: 50cc のみ Mario Cup と完全同一データ (speedFactor非ゼロ、behindSpd=-5、i13-i19=全0)。100cc/150cc は汎用テンプレートB。
3. **Wario Cup (c=3)**: 100cc のみ speedFactor 非ゼロ (0.2~0.6)。50cc/150cc は speedFactor=0。AI パターンは独自 (全CC同一)。
4. **DK Cup (c=2)**: speedFactor は全て0だが、50cc/100cc で aheadDist=15 (他コースは10)、behindSpd=5。AI パターンは独自 (全CC同一)。
5. **Bowser Cup (c=5)**: f02=50 (他は10)。50cc/100cc の GLD/ORI/ENH/V4 で timer が極端に短い (5~10)。AI パターンは独自。

### テンプレート分類

**テンプレートA (Pac-Man専用)**: AI パターンの i19=9。float パラメータは汎用と同一。
**テンプレートB (Rainbow/Yoshi 100+/Waluigi 共用)**: AI パターンの i19=5 or 6。i13-i15 はテンプレートAと同一だが i19 のみ異なる。
